Introduzione

Cos'è la dashboard

Il Engage Finance Commercial Monitor 2026 è una dashboard HTML standalone per il monitoraggio delle iniziative commerciali dell'area Finance: budget, ricavi, pipeline attiva e performance individuali dei commerciali.

Funziona interamente nel browser senza installazioni. Basta aprire il file HTML e caricare il proprio file Excel di monitoraggio per ottenere immediatamente grafici interattivi, KPI e tabelle filtrabili. I dati non vengono mai inviati a server esterni.

💡
Aggiornamento settimanale La dashboard è pensata per essere usata con rilasci periodici del file Excel (es. settimanali o mensili). Ogni nuovo file ha la data nel nome: caricando il file più recente come "principale" e quello precedente come "confronto", la sezione ⑦ calcola automaticamente tutte le variazioni.

File richiesto§ 2

File Excel richiesto

La dashboard legge un file Excel in formato .xlsx con una struttura specifica. Il nome del file deve seguire la convenzione con la data per consentire il riconoscimento automatico.

Convenzione del nome file

📁
Formato atteso Monitoraggio_commerciale_iniziative_di_budget_2026_YYYYMMDD.xlsx
La data nel nome (es. 20260415) viene estratta automaticamente e usata come data di riferimento per il calcolo delle urgenze e per l'etichetta "Rilevazione: GG/MM/AAAA" nell'header della dashboard.

Struttura del foglio

La dashboard legge il primo foglio del file Excel (SheetNames[0]). La prima riga valida con le intestazioni viene individuata automaticamente (funzione findHeaderRow). Le colonne vengono riconosciute per nome, non per posizione, quindi l'ordine delle colonne non è vincolante.

🔍
Riconoscimento automatico delle intestazioni La dashboard scorre le righe iniziali del foglio per trovare quella che contiene le intestazioni attese. Eventuali righe di titolo o metadati sopra l'header non causano errori, purché la riga di intestazione contenga almeno le colonne richieste.

Primo accesso§ 4

Schermata di caricamento

All'apertura del file HTML appare la schermata di caricamento. Supporta uno o due file Excel: il file principale (obbligatorio) e un file di confronto opzionale per l'analisi comparativa.

Procedura passo per passo

  1. 1
    File principale (obbligatorio) — trascina il file Excel più recente nell'area "📊 File principale" oppure clicca per selezionarlo. Il nome deve seguire la convenzione Monitoraggio_commerciale_..._YYYYMMDD.xlsx per l'estrazione automatica della data.
  2. 2
    File di confronto (opzionale) — se vuoi usare la sezione ⑦ "Confronto tra date", trascina qui il file della settimana precedente. I due file vengono elaborati separatamente e confrontati per rilevare nuove iniziative e variazioni di stato.
  3. 3
    Clicca "Genera Dashboard →" — il pulsante si attiva non appena almeno il file principale è caricato. L'elaborazione (parsing Excel + calcoli) avviene in pochi secondi direttamente nel browser.
🔄
Cambiare file Per ricaricare con un nuovo file, usa il pulsante ↺ Ricarica nella barra in alto. Questo riporta alla schermata iniziale e azzera tutti i dati in memoria. Non è necessario ricaricare la pagina dal browser.
📅
Data di riferimento e urgenza La data estratta dal nome del file principale diventa la data di riferimento per il calcolo dell'urgenza delle iniziative. Le iniziative con "Mese Inizio Ricavo" passato o entro 30 giorni dalla data di riferimento vengono marcate come 🔴 Critiche.

Tab ⑥§ 11

📉 KO & New Business

Tre sotto-sezioni dedicate alle iniziative perse, al new business non a budget e alla hot pipeline valorizzata ma non ancora finalizzata.

Iniziative KO — attività perse o bloccate

Elenco di tutte le iniziative con Stato della proposta = KO. Per ogni riga mostra:
  • Progetto, Cliente, Commerciale
  • Valore KO (col. N) — budget che si è perso
  • Tipo Ricavo e eventuali note
I KPI in cima mostrano il numero totale di iniziative KO e il loro valore budget aggregato. Utile per quantificare l'impatto delle perdite sul forecast annuale.

New Business — iniziative non a budget

Lista delle iniziative con flag Budget/New Business = "New Business": opportunità commerciali identificate durante l'anno che non erano incluse nel budget originale 2026.
  • New Business totali — numero di iniziative NB
  • New Biz OK — quante sono già finalizzate
  • New Biz Hot — quante sono in pipeline valorizzata
  • Copertura KO — quante delle KO sono parzialmente compensate dai NB

Hot Pipeline — potenziale valorizzato non confermato

Iniziative con col. T > 0 e Stato ≠ OK: proposte per cui esiste già un valore di ricavo atteso (col. T), ma che non sono ancora state formalmente finalizzate. È il valore "in bilico" che potrebbe diventare consuntivo.
  • Colonna Hot (S): importo della proposta
  • Colonna Ricavi 2026 (T): valore hot già valorizzato
  • Ordinabile per importo decrescente per identificare le opportunità di maggior valore

Concetti chiave§ 13

Urgenza & criticità

Il sistema di urgenza è uno degli elementi più operativi della dashboard. Segnala proattivamente le iniziative che rischiano di slittare o non essere finalizzate in tempo.

Come viene calcolata

  1. 1
    La dashboard legge la data di riferimento dal nome del file (es. 20260415 → 15 aprile 2026).
  2. 2
    Per ogni iniziativa non in stato OK, confronta il campo Mese Inizio Ricavo con la data di riferimento.
  3. 3
    Assegna il livello di urgenza: 🔴 Critica se l'avvio è già passato o entro 1 mese; 🟡 Alta entro 3 mesi; 🟢 Normale oltre 3 mesi.

Dove viene visualizzata

SezioneCome appare
① Executive OverviewBox riepilogativo con conteggio e budget totale delle 🔴 Critiche
② CommercialiGrafico "Iniziative Critiche per Commerciale" con n. e budget a rischio
⑤ PipelineColonna "Urg." nella tabella + filtri 🔴🟡🟢 in cima
💡
Iniziativa critica senza data di inizio Le iniziative prive del campo "Mese Inizio Ricavo" non ricevono un livello di urgenza e non vengono incluse nel conteggio delle critiche. Se stai notando un numero di critiche inferiore al previsto, verifica che tutte le righe abbiano la data di inizio compilata nel file Excel.

Supporto§ 15

Domande frequenti

❓
La dashboard non riconosce la data dal nome del file
La data di riferimento mostra la data odierna invece di quella del file
▼
Il nome del file deve contenere una sequenza di 8 cifre nel formato YYYYMMDD (es. 20260415). Se il file ha una data in formato diverso (es. 15-04-2026 o April2026), la dashboard non la riconosce e usa la data odierna come fallback. Rinomina il file prima di caricarlo.
❓
Alcune iniziative non appaiono in nessun pannello
Righe del file Excel che sembrano sparite
▼
Verifica che la colonna Stato della proposta sia compilata per quelle righe. Le righe con stato vuoto o con un valore non riconosciuto dalla dashboard (es. errori di digitazione) vengono saltate. Controlla anche che non ci siano righe vuote di separazione nel file.
❓
Il pannello ⑦ Confronto è vuoto o non appare
La sezione comparativa non mostra dati
▼
Il pannello ⑦ richiede che nella schermata di caricamento sia stato fornito sia il file principale sia il "File precedente — confronto". Se è stato caricato solo un file, la sezione confronto rimane vuota. Torna alla schermata iniziale con "↺ Ricarica" e carica entrambi i file.
❓
I valori numerici appaiono come 0 o come NaN
Budget, ricavi o hot pipeline mostrano valori errati
▼
Le colonne numeriche (N, S, T, U) devono contenere valori numerici puri nel file Excel. Cause comuni di errore:
  • La cella è formattata come testo (controllare il formato cella in Excel)
  • Il numero contiene il simbolo € o la virgola come separatore di migliaia
  • La cella contiene una formula con un errore (#VALORE!, #N/D, ecc.)
  • Il nome della colonna nel file non corrisponde esattamente a quello atteso
❓
Il numero di iniziative critiche sembra sbagliato
Meno (o più) critiche del previsto
▼
Verifica che: (1) il campo Mese Inizio Ricavo sia compilato per le iniziative interessate, (2) il nome del file contenga la data corretta nel formato YYYYMMDD — la data del file è la base per il calcolo, (3) le iniziative in stato OK non vengono conteggiate come critiche anche se la loro data di inizio è passata (il calcolo si applica solo alle non-OK).
❓
Come aggiornare la dashboard ogni settimana?
Workflow per l'aggiornamento periodico
▼
Il workflow consigliato è:
  • Esporta il nuovo file Excel con la data odierna nel nome
  • Apri la dashboard HTML nel browser
  • Nella schermata di caricamento, carica il nuovo file come "principale" e il file della settimana scorsa come "confronto"
  • Clicca "Genera Dashboard →"
  • La sezione ⑦ mostrerà automaticamente tutte le variazioni
Non è necessario modificare il file HTML: la dashboard si aggiorna leggendo i dati dal file Excel.
